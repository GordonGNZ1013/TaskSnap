【TaskSnap 數據存儲配置說明】

===========================================
📂 數據存儲路徑結構
===========================================

應用程序運行目錄下：
├── ProjectHelper.exe（或 TaskSnap.exe）
└── data/                                 ← 所有數據都存在這裡
    ├── tasks.db                          ← SQLite 數據庫（所有任務信息）
    ├── attachments/                      ← 上傳的附件
    │   ├── file1.pdf
    │   ├── file2.docx
    │   └── ...
    └── snapshots/                        ← 快照存檔（完成的任務快照）
        ├── Task_000001_2025-12-18_10-30-45.zip
        ├── Task_000002_2025-12-18_15-45-30.zip
        └── ...

===========================================
🎯 重要信息
===========================================

1. 所有數據都集中在應用程序目錄下的 data 文件夾
2. data 文件夾會在第一次運行時自動建立
3. 可以輕鬆備份：只需複製 data 文件夾即可
4. 可以輕鬆遷移：複製整個 data 文件夾到新位置

===========================================
📋 備份步驟
===========================================

方式 A：快速備份
  1. 進入應用程序目錄
  2. 複製 data 文件夾
  3. 粘貼到 D:\ 或 U: 盤

方式 B：壓縮備份
  1. 右鍵點擊 data 文件夾
  2. 發送到 → 壓縮文件夾
  3. 得到 data.zip 備份文件

===========================================
🔄 恢復數據步驟
===========================================

1. 關閉應用程序
2. 刪除應用程序目錄下的 data 文件夾
3. 從備份複製 data 文件夾到應用程序目錄
4. 重新啟動應用程序

===========================================
💡 遷移應用程序時的操作
===========================================

1. 複製整個應用程序目錄（包括 data 文件夾）到新位置
2. 或者只複製可執行文件，第一次運行會建立新的 data 文件夾

===========================================
📊 文件大小估計
===========================================

• 空數據庫：~100 KB
• 每個任務記錄：~1 KB
• 附件：根據實際文件大小
• 快照（ZIP 壓縮）：附件大小的 60-80%

示例：
  - 1000 個任務 + 100 個附件（平均 5 MB）
  - 預計 data 文件夾大小：~500 MB

===========================================
✅ 驗證配置是否成功
===========================================

1. 編譯並運行應用程序
2. 建立一個測試任務並上傳附件
3. 點擊「完成」建立快照
4. 打開文件管理器，進入應用程序目錄
5. 應該看到 data 文件夾及其子目錄
6. 確認快照 ZIP 文件已建立
